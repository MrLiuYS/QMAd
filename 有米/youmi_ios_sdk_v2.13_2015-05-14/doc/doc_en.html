3.Overall Setting Interface:

    // Setting appid and appSecret

    // We use synchronous method to request for online parameters

4.Interstitials Interface:

    // Initialize interstitials advertisement and the spot type of the interstitials (spot type can be lanscape, portrait or both)
    + (void)initYouMiDeveLoperSpot:(SpotType)spotType;

    // 'dismissAction' is the callback of closing advertisement
    + (BOOL)showYouMiSpotAction:(void (^)(BOOL flag))dismissAction;

    // The callback of clicking interstitials advertisement. If success to click, then flag is YES. Otherwise, flag is NO.
    + (BOOL)clickYouMiSpotAction:(void (^)(BOOL flag))callbackAction;

五.Example:

  • 1) Need to refer to this header file wherever invoking Youmi advertisement

    ConfigHeader.h

  • 2) Add Youmi advertisement initialization method in didFinishLaunchingWithOptions within file AppDelegate.m

    +(void)initYouMiDeveloperParams:(NSString *)YM_appid YM_SecretId:(NSString*)YM_SecretId;
Meanwhile, you can add initialization of interstitials advertisement
+(void)initYouMiDeveLoperSpot:(SpotType)spotType;

Three values for SpotType:
kSPOTSpotTypePortrait,
kSPOTSpotTypeLandscape,
kSPOTSpotTypeBoth,

If your application doesn't support rotation, and the application is used with portrait mode, then the value should be: kSPOTSpotTypePortrait,
If your application doesn't support rotation, and the application is used with landscape mode, then the value should be: kSPOTSpotTypeLandscape,
If your application supports rotation, then the value should be: kSPOTSpotTypeBoth.
(Notes: if your application doesn't support roation, and you still choose kSPOTSpotTypeBoth, the advertisement can also be shown, but we do not recommend it)

  • 3) Add the below code when it need to show advertisement

    + (BOOL)showYouMiSpotAction:(void (^)(BOOL flag))dismissAction;

  • 4) If you need event of click, you can add the callback of click. (This is optional)

    + (BOOL)clickYouMiSpotAction:(void (^)(BOOL flag))callbackAction;
